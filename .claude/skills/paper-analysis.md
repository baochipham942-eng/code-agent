# 论文解读 Skill

帮助用户快速理解学术论文的核心内容，提供结构化的解读报告。

## 使用场景

- 快速了解论文主题和贡献
- 提取论文的方法论和实验结果
- 生成论文摘要和阅读笔记
- 对比多篇论文的异同

## 解读流程

### 1. 读取论文

首先使用 `read_pdf` 工具读取论文内容：

```
read_pdf { "file_path": "paper.pdf" }
```

### 2. 结构化解读

按以下结构分析论文：

#### 基本信息
- **标题**:
- **作者**:
- **发表时间/会议/期刊**:
- **关键词**:

#### 研究问题
- 论文要解决什么问题？
- 为什么这个问题重要？
- 现有方法有什么不足？

#### 核心贡献
- 提出了什么新方法/理论/系统？
- 创新点是什么？（通常 2-4 点）

#### 方法论
- 技术路线概述
- 核心算法/架构
- 关键假设和约束

#### 实验验证
- 数据集和基准
- 评估指标
- 主要结果（用表格展示）
- 与 baseline 的对比

#### 局限性
- 作者承认的局限
- 潜在的问题或改进空间

#### 一句话总结
用一句话概括论文的核心贡献。

## 输出模板

```markdown
# 论文解读：{论文标题}

## 基本信息
| 项目 | 内容 |
|------|------|
| 标题 | {title} |
| 作者 | {authors} |
| 发表 | {venue} {year} |
| 领域 | {domain} |

## 研究动机
{1-2 段描述问题背景和动机}

## 核心贡献
1. {贡献1}
2. {贡献2}
3. {贡献3}

## 方法概述
{2-3 段描述方法}

## 实验结果
| 方法 | 指标1 | 指标2 |
|------|-------|-------|
| Baseline | x | y |
| 本文方法 | x' | y' |

## 局限与展望
{局限性和未来工作}

## 一句话总结
> {核心贡献的简洁表述}
```

## 注意事项

1. **忠于原文**：不要过度解读，引用论文原话时注明
2. **突出创新**：聚焦于与前人工作的区别
3. **批判思维**：适当指出论文的局限性
4. **可读性**：使用图表和结构化格式提高可读性
