## 一、项目概述

### 1.1 项目背景

Claude Code 作为 Anthropic 推出的 AI 编程助手，经历了从 v0.2 Beta 到 v2.0 的多次迭代，每个代际都带来了显著的能力提升。本项目旨在：

1. **学习研究** - 通过构建类似产品，深入理解 AI coding agent 的工作原理
2. **代际对比** - 可视化不同代际的能力差异，观察 prompt engineering 的演进
3. **实践探索** - 使用 DeepSeek 等开源模型验证相同架构的表现

### 1.2 项目名称

**Code Agent** - AI Coding Assistant

### 1.3 核心目标

| 目标 | 描述 |
|------|------|
| **功能复刻** | 实现 Claude Code 核心功能（文件操作、命令执行、代码搜索等） |
| **代际切换** | 支持切换 4 个主要代际的 system prompt，观察能力差异 |
| **交互体验** | 参考 Cowork 风格，提供友好的桌面应用界面 |
| **模型兼容** | 支持 DeepSeek API、Claude API 等多模型切换 |

## 二、Claude Code 代际演进分析

│  2025.02            2025.05             2025.06            2025.09      │

### 2.2 各代际详细能力矩阵

#### Generation 1: 基础工具期 (v0.2 Beta - 2025.02)

**核心特征**: 最小可用的编程助手

| 工具/能力 | 说明 |
|----------|------|
| `bash` | 执行 shell 命令 (git, npm, find, grep) |
| `read_file` | 读取文件内容 |
| `write_file` | 创建/覆盖文件 |
| `edit_file` | 精确编辑文件（基于字符串匹配） |
| `CLAUDE.md` | 项目记忆文件 |

**Prompt 特点**:
- 刚性规则为主 ("You MUST...", "NEVER...")
- 简单的工具调用指导
- 基础的安全约束

---

#### Generation 2: 生态融合期 (v1.0.x - 2025.05)

**核心特征**: 外部系统集成 + IDE 支持

| 工具/能力 | 说明 |
|----------|------|
| 继承 Gen 1 所有能力 | - |
| `MCP` (Model Context Protocol) | 外部系统集成协议 |
| IDE Integration | VS Code / JetBrains 集成 |
| Custom Slash Commands | `/commandname` 自定义命令 |
| GitHub Actions | CI/CD 集成 |
| `Glob` | 文件模式匹配搜索 |
| `Grep` | 内容搜索（基于 ripgrep） |

**Prompt 特点**:
- 更详细的工具使用说明
- MCP 服务器配置指导
- IDE 交互规范

---

#### Generation 3: 智能规划期 (v1.0.60+ - 2025.06)

**核心特征**: 多 Agent 编排 + 规划能力

| 工具/能力 | 说明 |
|----------|------|
| 继承 Gen 1-2 所有能力 | - |
| `Plan Mode` | 执行前规划模式 (Shift+Tab) |
| `Task` (Subagents) | 子代理系统，专门化任务分发 |
| `TodoWrite` | 任务追踪与进度管理 |
| `/context` | 上下文窗口可视化 |
| `/agents` | 代理管理命令 |
| `AskUserQuestion` | 交互式问题工具 |
| `MultiEdit` | 批量文件编辑（后移除） |

**Prompt 特点**:
- 引入规划思维指导
- 子代理调用规范
- 任务分解策略

---

#### Generation 4: 工业化系统期 (v2.0.x - 2025.09+)

**核心特征**: 完整的插件生态 + 高级自动化

| 工具/能力 | 说明 |
|----------|------|
| 继承 Gen 1-3 所有能力 | - |
| `Skills` (Agent Skills) | 可移植能力包系统 |
| `Skill` Tool | 技能执行工具 |
| `Hooks` | 事件驱动自动化（11 种事件类型） |
| `Plugins` | 完整插件系统 + Marketplace |
| `LSP` Integration | Language Server Protocol 集成 |
| `WebFetch` / `WebSearch` | 网络访问能力 |
| `NotebookEdit` | Jupyter Notebook 编辑 |
| Named Sessions | 会话持久化与恢复 |
| Async Agents | 异步代理执行 |
| `EnterPlanMode` / `ExitPlanMode` | 显式规划模式控制 |

**Prompt 特点**:
- 从刚性规则转向柔性指导
- 更信任模型的自主判断
- 完善的 Git 安全协议
- 详细的工具使用优先级指导

#### 3.1.2 工具系统 (按代际分层)

**Gen 1 基础工具**:

| 工具 | 参数 | 功能 |
|------|------|------|
| `glob` | `pattern`, `path?` | 文件模式匹配 |
| `grep` | `pattern`, `path?`, `include?` | 内容搜索 |
| `list_directory` | `path` | 列出目录内容 |

**Gen 3 智能工具**:

| 工具 | 参数 | 功能 |
|------|------|------|
| `task` | `prompt`, `subagent_type` | 创建子代理任务 |
| `todo_write` | `todos[]` | 管理任务列表 |
| `ask_user_question` | `questions[]` | 交互式询问 |

**Gen 4 高级工具**:

| 工具 | 参数 | 功能 |
|------|------|------|
| `skill` | `skill_name`, `args?` | 执行技能 |
| `web_fetch` | `url`, `prompt` | 获取网页内容 |
| `web_search` | `query` | 网络搜索 |

## 四、非功能需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| 首次启动 | < 3 秒 |
| 消息响应 | 流式输出，首字符 < 500ms |
| 文件操作 | < 100ms (常规大小文件) |
| 内存占用 | < 500MB (空闲状态) |

| 平台 | 版本要求 |
|------|----------|
| macOS | 11.0 (Big Sur) 及以上 |
| Windows | Windows 10 及以上 (后续支持) |
| Linux | Ubuntu 20.04 及以上 (后续支持) |

### 4.3 安全要求

- 文件操作需用户确认
- 敏感命令 (rm -rf, git push --force) 需二次确认
- API Key 本地加密存储
- 工作目录沙箱隔离
