# ADR-043：组级工具步骤三态折叠预览（T2）

- 状态：已拍板（2026-07-17，产品负责人逐条确认）
- 来源：Grok Build 借鉴 TUI 批 T2 条（降档并入 UI/UX 审查批）
- 影响面：聊天转录里的工具步骤组（ToolStepGroup），独占施工面

---

## 一、背景与现状（2026-07-17 实核，修正工单两处过时描述）

Agent 干活时连续的工具调用在转录里折成一行组。折叠目前只有两个档位：收起（一行全盲）↔ 全部展开（刷屏）。

**实核修正 ①**：工单写"流式 turn 默认全展开=刷屏"已过时——2026-05-08（`d19b912cd4`）把默认改成了永远收起。现状是流式期**一行全盲**：agent 干活时用户只看到"运行中"，任何进度都要手动点开，点开又刷屏。两头都不对，这正是三态中间档要填的坑。

**实核修正 ②**：组件现已有产品拍板过的失败分级：需用户介入的失败（鉴权/额度/限流）强制展开醒目；探索性失败（agent 试错）保持收起、中性色。三态设计与它对账见决策 5。

**已有的轮子（复用，不新造）**：

| 已有能力 | 位置 |
|---|---|
| 单工具头尾截断预览（运行中尾 5 行 / 完成后头 4+尾 16） | `bashOutputPreview.ts` |
| 运行中增量输出通路（`tool_output_delta`→`liveOutput`） | `useToolExecutionEffects.ts` |
| 单工具行运行中有实时输出自动展开、成功自动收起 | `ToolCallDisplay/index.tsx` |
| 需介入失败强制展开 | `ToolStepGroup` 的 `forceExpandOnFailure` |

参考对象：Grok Build TUI 三态 Collapsed/Truncated/Expanded；其 TUI 是社区口碑最硬一块，感知性能 ROI 高。

范围澄清：消息气泡路径的旧组折叠组件（`ToolCallGroup`）主视图未使用，不在本次范围。

## 二、拍板决策（2026-07-17 全部按推荐通过）

1. **流式期默认 = 中间档（Truncated）**：自动透出正在跑的那一步的实时尾部输出（复用单工具尾 5 行预览），已完成步骤收成一行计数。不点也能看到进度，不刷屏。
2. **历史 turn 默认 = 收起（维持现状）**：一行摘要；流式收尾时中间档自动降回收起；用户手动动过档位的不抢（冻结自动切换）。
3. **交互 = 两态开关 + 自动中间档**：点击/键盘仍是「收起 ↔ 全部展开」，中间档是流式期系统自动档，不进手动循环。aria-expanded 保持布尔语义（中间档 = 未展开），不破坏 Batch 1 键盘协议。
4. **内容配额 = 全部复用现成**：中间档 = 组标题行 + 正在运行的那步（现成尾 5 行预览）+ 一行「已完成 N 步」计数。组级唯一新文案是计数行，进 i18n 域级子文件。不另设配额系统。
5. **失败不走中间档（维持现状拍板）**：需介入失败直接跳全部展开并醒目（现有逻辑原样）；探索性失败保持收起中性（现有拍板原样）。中间档只服务"正常干活中"。

## 三、施工要点

- 组件内部档位推导：用户/失败强制展开 → 全量；流式中且用户未手动操作 → 中间档；其余 → 收起。流式结束自动回收起，userToggled 冻结自动档。
- 复用 `ToolCallDisplay` 紧凑行渲染运行中那步（无 result 即运行中），实时预览走其已有 liveOutput 通路，零新输出折叠代码。
- 新词条进 `src/renderer/i18n/chatTranscript.ts`（zh/en 相邻导出双侧同步）；组件在 chat CJK 棘轮 MIGRATED 清单，裸中文当场红。
- 不破坏：aria-expanded 布尔语义、Batch 1 键盘协议、探索性失败中性化、组头摘要去重（P0 #1）、forceExpandOnFailure。
- 测试：TDD；mock useI18n（先例 `turnDiffSummary.confirmation.test.tsx`）；监工变异验收。
- Dogfood：Dev 测试包 8181 + `CODE_AGENT_DISABLE_RENDERER_HOT_UPDATE=1` 验流式观感。

## 四、不做什么

- 不动消息气泡路径旧组折叠组件；不新造输出截断系统；不加秒表/计时动效（反焦虑拍板）；不做组级配额配置项（YAGNI）。
