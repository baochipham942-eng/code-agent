Neo 把 MCP 做「深」：scope 分层、权限刹车、长时运行健壮性 —— 都是自动改文件/执行命令的编程 Agent 刚需。
DeepChat 把 MCP 做「广」：反向暴露 + ACP + UI 渲染 —— 服务于「模型无关」通用客户端的生态野心。

2Computer Use 能力对比

桌面 GUI 自动化 —— 截图/坐标操控整台机器。两家都有，但取向不同。
维度NeoDeepChat胜
来源argus（源自 Anthropic Chicago）+ 内置 computer_usevendor 自 trycua / cua.ai 开源驱动，非自研都靠整合
工具粒度argus 27 工具（click 5 种/拖拽/键盘/批量/teach 教学）~30 工具（含 set_value/page/zoom/replay 轨迹回放/agent cursor）平
OS 支持macOS + Windows（Linux 不支持）仅 macOS（platforms:[darwin]）Neo
交互策略以像素坐标为主 + coordSpace=image 缩放 + AX 路径（可选）AX 可访问性树优先：get_window_state 取元素索引→操作；像素是最后兜底DeepChat 更先进
降级阶梯坐标 / 截图 AI 分析AX 树 → Electron AX 强启 → CDP DOM(9222) → 截图 → zoom → 像素DeepChat
后台操作macOS CGEvent 可后台（无需前台）前台 + helper app（DeepChat Computer Use.app）Neo
权限粒度注解分级 + 规则/LLM 分类器per-tool ask/allow 写死 plugin.json（读 allow·动作 ask）各有体系
分发/产品化env 开关（CODE_AGENT_ENABLE_ARGUS_MCP=1），默认关签名官方 .dcplugin + agent cursor 浮层 + 遥测DeepChat
Neo：双平台 + 后台 CGEvent + 工具粒度细，覆盖与工程纵深更强；但点击仍以像素坐标为主。
DeepChat：mac 单平台，但「AX 树优先取元素索引、像素仅兜底」的策略明显更先进，且对 Electron/浏览器走 CDP DOM，封装成签名插件、产品化最好。

3Browser Use 能力对比

浏览器自动化 —— 这一项 Neo 明显领先。
维度NeoDeepChat胜
有无✓ 内置 browser_action（25 动作）+ 可选 Playwright/Puppeteer MCP部分：inMemory 网页检索 + 应用内 BrowserPanelNeo
引擎Playwright（Chromium）· 托管浏览器池 · CDP 远程内置 web 访问 + UI 浏览面板Neo
交互方式browser-use 式：get_dom_snapshot / get_a11y_snapshot / targetRef / smart_click（+可选截图兜底）无独立浏览器引擎；靠 CUA 把浏览器当 mac App，必要时开 CDP(9222) 够 DOMNeo
内容获取DOM 抽取 + HTML + 可访问性树 + 截图web information retrieval（抓取/检索式）Neo
会话隔离多 Agent 独立浏览器会话池 + storageState 导入导出多会话并行（对话层，非浏览器自动化）Neo
默认启用✓ browser_action 默认开web 检索开箱即用；交互自动化缺位Neo
Neo 在 Browser Use 上是降维优势：有专门的 Playwright 工具集，走「抽 DOM/a11y → targetRef 按元素操作」的 browser-use 正路（快、准、省 token），另保留截图兜底。
DeepChat 此项偏弱：网页能力主要是「检索/抓取 + 让 CUA 把浏览器当普通桌面 App 点」，没有结构化的交互式浏览器自动化层。（注：基于公开 README/commit，未排除闭源插件中另有实现）

4三项能力总览 & 给 Neo 的启发

能力NeoDeepChat
MCP做「深」：6 层 scope + 权限刹车 + 健壮性做「广」：反向暴露 + ACP + UI 渲染
Computer Use双平台 · 27 工具 · 后台 CGEvent · 像素为主mac · trycua 驱动 · AX 树优先策略更先进 · 插件化
Browser Use专用 Playwright 层，browser-use 正路无独立引擎，靠 CUA+CDP 够 DOM（mac）
一句话：Neo 是「双引擎全栈 + 双平台」——computer use（覆盖整机）与 browser use（专用 Playwright，DOM 式）都齐，覆盖面与浏览器自动化领先；DeepChat 聚焦 mac computer use，靠整合 trycua 把「AX 树优先」策略做得比 Neo 更聪明、产品化更好，但浏览器自动化与跨平台是其短板。
给 Neo 的动作来源优先级
CUA 引入「优先 element-index、稀疏 UI 才降级视觉」混合策略抄 DeepChat 的点击策略，提准降本高
把 computer use 封装成签名插件、按需分发抄 DeepChat 的产品化封装中
反向暴露 Neo 能力为 MCP server抄 DeepChat 的「模型无关」杠杆中
守住 Browser Use + scope + 权限三大护城河这是相对 DeepChat 的结构性优势守
