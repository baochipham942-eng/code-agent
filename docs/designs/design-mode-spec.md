# Design Mode 产品 Spec（设计工作区）

> **状态**：v1 + T1-T6 已交付（T1-T6 已合 main，PR #258，merge `b377aa424`；前序 Cowart 画布 PR #257）。本文是产品层总览；技术架构见 `docs/architecture/design-mode.md`，画布深度设计与决策见 `docs/designs/design-canvas-cowart.md`，竞品来源见 `docs/competitive/kun-设计tab-借鉴清单.md` 与 `docs/competitive/opendesign-lovart-借鉴清单.md`（T1-T6 借鉴源）。

---

## 1. 一句话

> 在 Neo 里开一个与「通用(Code)」并列的顶层「设计」工作区，让用户**从需求直接产出设计物**——可交互 HTML 原型、AI 设计稿/信息图，并能在无限画布上**圈选区域精确局部重绘（带一致性锁定）+ 扩图/去水印 + 非破坏性版本对比 + 成本可见 + 导入自有图迭代**。

## 2. 问题与目标

- **问题**：编程 Agent 只能产代码；设计类需求（落地页、UI 稿、信息图、原型）此前无承载入口。
- **目标**：把"需求 → 设计物"做成一等流程，复用已有的 agent loop / 模型路由 / 图像生成基建，新增逻辑独立成模块。
- **竞品参照**：Kun(DeepSeek-GUI) 的设计 tab（原型 + 出图 + 反 AI 痕迹自检）打底；Liblib 星流(Cowart) 的无限画布交互范式做对照，并补其软肋；OpenDesign + Lovart 的「信任 UX + 确定性控制 + anti-slop」要素提炼成 T1-T6 落地（借鉴清单见 `docs/competitive/opendesign-lovart-借鉴清单.md`）。

## 3. 用户与场景

- **主用户**：用 Neo 做产品/设计探索的人（PM、设计、独立开发）。
- **典型场景**：① 一句话生成落地页可交互原型 ② 出一张 UI 设计稿/竖版信息图 ③ 把生成图或**自己上传的图**框选某块做局部修改 ④ 多版并排选定主版 ⑤ 导出。

## 4. 三条产品路径

| 路径 | 产物 | 交互 | 状态 |
|------|------|------|------|
| **交互原型** | 单文件可交互 HTML | 填需求 →（T5 首轮强制澄清表单 + 方向卡）→ 生成 → iframe 实时预览（边生成边长出来）；T6 运行时一键换肤试色 | ✅ |
| **设计稿 / 信息图** | 静态图像 | 填需求 + 品牌色/语气/**尺寸** →（T2 出图前显示预估 ¥）→ 生成 → 图铺上**无限画布** | ✅ |
| **画布迭代** | 图像 | 圈选区域 + 指令 → **真·局部重绘**（框外不动，T4 一致性锁定逐像素保证）→ 新版带血缘 → **非破坏性版本对比**定主版 | ✅ |
| **扩图 / 去水印** | 图像 | 选图 → 选方向(四向/全向)+比例 **扩图**补绘 / 一键**去文字水印** → 新版挂 spine | ✅ T3 |
| **自由画布** | 图像 | **导入**自有图（按钮/⌘V 粘贴/拖拽）→ 选中即可圈选重绘 | ✅ |

横切：**设计质量自检**——agent 写出的前端源码被自动扫"AI 痕迹"（紫蓝渐变/米色底/弹跳缓动/灰图床占位…），advisory 回注让模型自我修正（前置 prompt 约束 + 事后自检双保险；T6 新增禁灰框、劝用 `picsum.photos/seed` 真图规则）。

## 5. 差异化（vs Liblib 星流）

星流把"选区编辑"降级、靠 agent 自动定位元素，导致实测**选区精度弱、无法精确改某块、无结构化对比**。Neo 反向把这些做成一等公民：

- **精确圈选 + 区域级 mask ground 到模型**（通义万相 `wanx2.1-imageedit`，框外像素不动；T4 再加一致性锁定逐像素硬保证）
- **结构化版本对比**（等高并排 + 设为主版/淘汰，而非画布上人眼扫；T1 升级为非破坏性 spine，永不覆盖）
- **血缘版本树**（每次重绘记 `parentId`）

## 5.5 OpenDesign + Lovart 借鉴落地（T1-T6，已交付）

> 提炼自 `docs/competitive/opendesign-lovart-借鉴清单.md`：抓「信任 UX + 确定性控制 + anti-slop」三类，避开慢 agent 回路。T1→T2 串行打底，T3-T6 并行各挂 spine。

| # | 能力 | 用户价值 | 状态 |
|---|------|---------|------|
| **T1** | 非破坏性版本 spine | 每步操作落新版永不覆盖，淘汰是软删可还原；canvas + proto 共用同一对比浮层（设主版/淘汰落盘）| ✅ |
| **T2** | 成本透明 + undo/redo | 出图/重绘前显示预估 ¥（0.14/张），历史每步可命名、显示实际花费 + 累计，一键回滚到前一版 | ✅ |
| **T3** | 扩图 + 去水印 | 选图按方向(四向/全向)+比例扩展画面，或一键去文字水印（通义万相原生能力）| ✅ |
| **T4** | 一致性锁定再编辑 | 局部重绘「只改这块、其余逐像素不动」的硬保证（模型偷改的未选区自动贴回原图 + diff 证据图）| ✅ |
| **T5** | 方向卡 + 参考截图入口 | 成品向产物首轮强制澄清表单（surface + 3 张方向卡：色板/字体/调性/参考），或选「匹配一张参考截图」分支，嫌烦可「直接生成」逃生 | ✅ |
| **T6** | 运行时换肤 + seed 真图 | 原型预览态零重生成一键切 5 套色板试色；新原型用 `picsum.photos/seed` 真图占位而非灰框（lint 强制）| ✅ |

> T6 换肤仅作用于交互原型（proto HTML 预览）；画布静态 PNG 是像素图，换肤套不进去（故不做）。

## 6. 范围

**v1 已交付**：顶层 Code/设计导航 · 三产物类型 · 原型实时预览 · 画布(konva)出图回灌 · 圈选真 inpaint · 版本对比 · 自由画布导入 · 尺寸选择 · 图片导出 · 设计质量自检 hook。

**T1-T6 已交付**（PR #258）：非破坏性版本 spine · 成本透明 + undo/redo · 扩图/去水印 · 一致性锁定再编辑 · 方向卡 + 参考截图入口 · 运行时换肤 + seed 真图占位。

**仍不做 / 后续**：agent 自主多轮编辑（让 agent 自己看结果再改）· 工作流历史链可视化 · 原型高级特性（设备切换/圈选改/版本快照，独立分支）· 桌面包真机 dogfood。

## 7. 关键决策（拍板记录，详见 canvas spec）

| # | 决策 | 结论 | 理由 |
|---|------|------|------|
| D1 | 画布库 | konva 自研（非 tldraw）| 规避 tldraw 商用 $6k/年或水印；MIT 免费、能力够 |
| D2 | 图像引擎 | 通义万相（DashScope）| 用户已有百炼 key、免代理、原生 mask inpaint、base64 直传 |
| D3 | v1 范围 | 核心闭环 + A/B 对比 | 差异化优先 |
| T1 | 版本模型 | 非破坏性 variant spine（永不覆盖，软删保留）| 信任 UX——任何操作可还原；canvas 与 proto 统一模型 |
| T4 | 编辑一致性 | region-lock + diff-gate（未选区贴回原图）| 扩散 inpaint 会系统性偷改 mask 外区域；逐像素锁死才算「只改这块」|
| T5 | 澄清前置 | 成品向首轮强制方向卡表单 + 逃生口 | anti-slop 最低门槛；逃生口尊重用户选择 |

**成本提醒**：出图/重绘按次计费（通义万相文生图便宜、局部重绘/扩图约 0.14 元/张，价表唯一源 `shared/constants/pricing.ts`），仅用户显式触发；T2 起出图前显示预估 ¥ + 历史显示实际花费。

## 8. 成功标准 / 验证

- 各路径真实端到端验证（headless 驱动 + 截图）：文生图落画布、圈选 inpaint 框外不动、版本对比落盘主版、导入自有图可重绘；T3 扩图/去水印出新版、T4 越界自动贴回 + diff 证据、T5 首轮强制表单回流 brief、T6 换肤零重生成 + 导出不含注入样式。
- 质量：typecheck=0、设计相关单测全绿（T1-T6 各模块独立单测，含 variantSpine/protoSpine/variantHistory/imageConsistency/imageCost/designPreviewInject/questionFormPreview…）、CogView/FLUX/PPT 零回归、全程 i18n、禁硬编码（价格/常量入 `shared/constants`）、新增 IPC 同步登记 shellCapabilities。
