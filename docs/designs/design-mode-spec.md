# Design Mode 产品 Spec（设计工作区）

> **状态**：v1 已交付（PR #257）。本文是产品层总览；技术架构见 `docs/architecture/design-mode.md`，画布深度设计与决策见 `docs/designs/design-canvas-cowart.md`，竞品来源见 `docs/competitive/kun-设计tab-借鉴清单.md`。

---

## 1. 一句话

> 在 Neo 里开一个与「通用(Code)」并列的顶层「设计」工作区，让用户**从需求直接产出设计物**——可交互 HTML 原型、AI 设计稿/信息图，并能在无限画布上**圈选区域精确局部重绘 + A/B 对比 + 导入自有图迭代**。

## 2. 问题与目标

- **问题**：编程 Agent 只能产代码；设计类需求（落地页、UI 稿、信息图、原型）此前无承载入口。
- **目标**：把"需求 → 设计物"做成一等流程，复用已有的 agent loop / 模型路由 / 图像生成基建，新增逻辑独立成模块。
- **竞品参照**：Kun(DeepSeek-GUI) 的设计 tab（原型 + 出图 + 反 AI 痕迹自检）打底；Liblib 星流(Cowart) 的无限画布交互范式做对照，并补其软肋。

## 3. 用户与场景

- **主用户**：用 Neo 做产品/设计探索的人（PM、设计、独立开发）。
- **典型场景**：① 一句话生成落地页可交互原型 ② 出一张 UI 设计稿/竖版信息图 ③ 把生成图或**自己上传的图**框选某块做局部修改 ④ 多版并排选定主版 ⑤ 导出。

## 4. 三条产品路径

| 路径 | 产物 | 交互 | 状态 |
|------|------|------|------|
| **交互原型** | 单文件可交互 HTML | 填需求 → 生成 → iframe 实时预览（边生成边长出来）| ✅ |
| **设计稿 / 信息图** | 静态图像 | 填需求 + 品牌色/语气/**尺寸** → 生成 → 图铺上**无限画布** | ✅ |
| **画布迭代** | 图像 | 圈选区域 + 指令 → **真·局部重绘**（框外不动）→ 新版带血缘 → **A/B 对比**定主版 | ✅ |
| **自由画布** | 图像 | **导入**自有图（按钮/⌘V 粘贴/拖拽）→ 选中即可圈选重绘 | ✅ |

横切：**设计质量自检**——agent 写出的前端源码被自动扫"AI 痕迹"（紫蓝渐变/米色底/弹跳缓动…），advisory 回注让模型自我修正（前置 prompt 约束 + 事后自检双保险）。

## 5. 差异化（vs Liblib 星流）

星流把"选区编辑"降级、靠 agent 自动定位元素，导致实测**选区精度弱、无法精确改某块、无结构化对比**。Neo 反向把这些做成一等公民：

- **精确圈选 + 区域级 mask ground 到模型**（通义万相 `wanx2.1-imageedit`，框外像素不动）
- **结构化 A/B 版本对比**（等高并排 + 设为主版/淘汰，而非画布上人眼扫）
- **血缘版本树**（每次重绘记 `parentId`）

## 6. 范围

**v1 已交付**：顶层 Code/设计导航 · 三产物类型 · 原型实时预览 · 画布(konva)出图回灌 · 圈选真 inpaint · A/B 对比 · 自由画布导入 · 尺寸选择 · 图片导出 · 设计质量自检 hook。

**v1 不做 / 后续**：agent 自主多轮编辑（让 agent 自己看结果再改）· 工作流历史链可视化 · 原型高级特性（设备切换/圈选改/版本快照，独立分支）· 桌面包真机 dogfood。

## 7. 关键决策（拍板记录，详见 canvas spec）

| # | 决策 | 结论 | 理由 |
|---|------|------|------|
| D1 | 画布库 | konva 自研（非 tldraw）| 规避 tldraw 商用 $6k/年或水印；MIT 免费、能力够 |
| D2 | 图像引擎 | 通义万相（DashScope）| 用户已有百炼 key、免代理、原生 mask inpaint、base64 直传 |
| D3 | v1 范围 | 核心闭环 + A/B 对比 | 差异化优先 |

**成本提醒**：出图/重绘按次计费（通义万相文生图便宜、局部重绘约 0.14 元/张），仅用户显式触发。

## 8. 成功标准 / 验证

- 四条路径均真实端到端验证（headless 驱动 + 截图）：文生图落画布、圈选 inpaint 框外不动、A/B 对比落盘主版、导入自有图可重绘。
- 质量：typecheck=0、40 设计相关单测全绿、CogView/FLUX/PPT 零回归、全程 i18n、禁硬编码。
