# Agent Neo v0.16.101

发布日期：2026-06-10

## 主要变更

### Intel Mac (x64) 双架构首发
- 正式支持 Intel Mac：发版矩阵 arm64（macos-latest）+ x64（macos-15-intel）并行构建，各自原生 rebuild native 模块（better-sqlite3/keytar），sidecar（rtk/uv/Swift 三件套）按架构产出。
- 单 manifest 双平台键：`stable/latest.json` 同时含 `darwin-aarch64` + `darwin-x86_64`，Tauri updater 按自身架构选包；in-app 更新检查请求携带 arch。
- 分发页下载按钮按芯片选包（WebGL 启发式识别 Apple Silicon/Intel + 手动切换兜底）；`/api/update` 按 `arch` 路由 dmg。
- x64 已知限制：VAD 语音活动检测不适配（onnxruntime-node 无 darwin-x64 包），自动静默降级，其余功能与 arm64 对齐。

### Computer Use 新底座（CUA，默认关闭）
- cua-driver (trycua, MIT) 重签为「Agent Neo Computer Use.app」（自有 bundle id，TCC 授权弹窗显示 Agent Neo），stdio MCP 接入，AX 树优先 + 后台不抢焦点。
- 权限 UI：Accessibility 必需 + 一键授权，录屏可选；用户菜单新增「桌面操作」入口。
- CUA 重签产物走发版 OSS 预构建分发（sha256 供应链锁定），CI 不依赖本机源。
- 默认 `CODE_AGENT_ENABLE_CUA=1` 才启用，开启时旧 computer_use 工具让位。

### 模型与中转站体验
- 修复「测试连接成功但会话 404」：baseUrl 末尾斜杠在解析层统一 trim（此前测试链路 trim 而 chat 链路裸拼，配置末尾带 `/` 时拼出 `//chat/completions`）。
- HTTP 404 错误信息带实际请求 URL + `/v1` 提示；aiSdk 路径对 HTTP≥400 落带 URL 的本地日志，statusText 一句 "Not Found" 不再无从排查。
- 视觉分析改走智能候选路由 + 失败引导；默认视觉模型对齐默认 provider（mimo-v2-omni）。

### 会话与诊断
- 会话右键菜单新增「导出会话日志」：脱敏诊断包 + 当天本地日志尾部，未登录用户也能一键导出给开发者排查。
- 会话导出（Markdown/日志）改用原生「另存为」对话框（Tauri 模式），用户自选保存位置。
- 修复工具结果落库前 eager 压缩导致的模型重试循环：新鲜 observation 原样可见，大文本统一由 L1 投影层处理（跨轮幂等重截断）。

### 稳定性
- 根治 WKWebView 启动连刷：启动 URL 加每次唯一 `?boot=` 参数 + index.html 响应 `Cache-Control: no-store`。
- MCP 认证失败路由到重新授权流程。
- 过渡轮 thinking 收纳进工具聚合组，不再打断时间线。

## 升级说明
- arm64 存量用户：自动更新无感升级（OSS key 兼容保留）。
- Intel 用户：首次安装从分发页下载 x64 DMG；后续自动更新走 `darwin-x86_64` manifest 键。
