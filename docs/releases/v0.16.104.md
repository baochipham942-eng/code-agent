# Agent Neo v0.16.104

发布日期：2026-06-12

## 主要变更

### Agent 运行时可靠性
- **运行时护栏更硬**：新增 doom-loop guard、Task gate、max-step 兜底、goal impossible 止损、retry 分类和 provider 失败友好提示，减少模型卡循环、误报完成或把底层错误直接甩给用户的情况。
- **Checkpoint 恢复不卡前台**：checkpoint writer 保持后台 LLM 子代理路径，但聊天主循环只短等前台窗口；超时或没有明确成功记录时会回到 summary 压缩，避免长时间卡住或读到旧 checkpoint。
- **嵌套子代理可用**：子代理可以继续委派子代理，整棵树共享深度、并发、超时和 token 预算，取消和孤儿回收按子树处理。

### 经验沉淀与记忆
- **History / Dream / Distill 主链路补齐**：transcript FTS、History 工具、memory BM25、dream consolidation 和 `/distill` 经验沉淀串成可验证路径。
- **Skill 草稿更克制**：自动蒸馏出的 skill 只进入待确认草稿，不会自动安装；`grep-read-edit` 这类只描述工具序列的低价值名字会被直接拒绝。

### 运维与发版验收
- **Renderer 热更新更安全**：active renderer bundle 版本低于当前 shell version 时会回退 builtin renderer，避免旧前端遮住新壳修复。
- **生产 verifier 不再无限等**：renderer hot-update 生产验收增加 metadata / bundle 下载超时和阶段日志，control-plane、manifest、release-record、app update 哪一步卡住会直接暴露。
- **MCP 与 admin 面补齐**：普通登录用户可自助管理 MCP server；HTTP Streamable MCP 支持 `url` alias 和 headers；管理员可在 dashboard 里授予或撤销他人 admin。

## 修复
- 修复 dream 弱证据候选可能写入长期记忆的问题。
- 修复 prompt variant A/B 归因缺 metadata、无内容 prompt version bump 污染 telemetry 的问题。
- 修复 weekly cron interval 在运行时不支持但 UI 仍展示的问题。
- 修复 vision 分析空响应被误报为 generic exception 的问题。
- 修复 session task tree 恢复时丢失 `parent_task_id` 的问题。
