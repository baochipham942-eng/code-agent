### 3. LocalProvider Tool Calling 动态判断

LocalProvider 根据模型 `supportsTool` 属性决定是否传工具定义：
- `supportsTool === true` → 发送 OpenAI 格式 tools + function calling
- `supportsTool !== true` → 降级为纯文本消息（`convertToTextOnlyMessages`）

VolcengineProvider 同理，但判断逻辑反向：`supportsTool !== false` 则传工具（默认支持）。

相关代码：
- `src/main/model/providers/localProvider.ts` — `modelInfo?.supportsTool === true`
- `src/main/model/providers/volcengineProvider.ts` — `modelInfo?.supportsTool !== false`

### 4. Provider 连通性测试

3 种协议策略适配不同 API 接口，返回延迟 + 结构化错误建议：

| Provider 类型 | 测试方式 |
|---------------|---------|
| Claude | `POST /messages` 最小请求 |
| Gemini | `GET /models` |
| OpenAI 兼容 | `GET /models` |

返回 `TestConnectionResult`：`{ success, latencyMs, error?: { code, message, suggestion } }`

相关代码：`src/main/ipc/provider.ipc.ts` — `buildTestConfig()` + `testConnection()`

### 5. Provider Health Monitor

事件驱动的四状态机，纯被动记录无后台线程：

```
healthy ──[errorRate≥30%]──→ degraded ──[errorRate≥70%]──→ unavailable
   ↑                                                          │
   └──────[3次连续成功]──── recovering ←──[任意成功]───────────┘
```

- 延迟滑动窗口（最近 10 次请求）→ P50/P95
- 错误率统计（5 分钟窗口）→ errorRate
- `getHealth(provider)` 返回 `ProviderHealth`，供 ModelSwitcher 展示

相关代码：`src/main/model/providerHealthMonitor.ts`

### 6. ModelSwitcher UX 增强

- 搜索框 — 实时过滤模型列表
- 能力标签 — vision/tool/reasoning 图标标识（Eye/Wrench/Brain）
- 模型全名显示 — `getModelDisplayLabel()` 替代 raw model ID
- Provider 健康色点 — 连接 ProviderHealthMonitor 状态

相关代码：`src/renderer/components/StatusBar/ModelSwitcher.tsx`
