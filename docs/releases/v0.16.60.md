# v0.16.60 — Hooks 成熟化 + 控制面板 + 会话连续性

四条并行 Stream：Hooks 系统从实验性推进到生产可用（事件审计 → observer 模式 → 完整接线 → 可观测性），同时补齐命令行控制面板和会话连续性。

## Stream A: Hooks 系统成熟化（6 commits）

### 1. 事件审计与清理

移除 3 个死事件（FileChanged / CwdChanged / ConfigChange），为所有存活事件标注稳定性级别：

| 级别 | 事件数 | 说明 |
|------|--------|------|
| stable | 9 | PreToolUse, PostToolUse, Stop 等核心事件 |
| experimental | 2 | SubagentStart, SubagentStop |
| planned | 2 | TaskCreated, TaskCompleted |
| internal/deprecated | 2 | 仅内部使用 |

相关代码：`src/main/hooks/events.ts`

### 2. Observer 模式

新增 `hookType` 配置字段（`'decision'` | `'observer'`）：

- **decision** — 默认类型，可阻止或修改工具执行
- **observer** — 正常执行并接收完整上下文，但 block/modify 结果被静默忽略

Observer-only 事件（PostToolUse, PostExecution, SessionStart 等）自动将 decision hook 降级为 observer 并发出警告。

相关代码：
- `src/main/hooks/configParser.ts` — hookType 解析 + 自动降级
- `src/main/hooks/merger.ts` — observer 优先合并策略
- `src/main/hooks/hookManager.ts` — observer 语义强制

### 3. Planning → User Hook 桥接

HooksEngine（规划系统内部）的 pre/post tool use 和 stop 事件现在 fire-and-forget 转发到 HookManager，使规划模式触发的工具执行对用户配置的 hook 可见。

相关代码：`src/main/planning/hooksEngine.ts`

### 4. 事件接线完成

三组之前有定义但从未被调用的事件全部接线：

**Part A — SubagentStart / SubagentStop**:
- 创建 `triggerSubagentStop()`（之前缺失）
- `SubagentContext` 接口新增 hookManager
- 在 `subagentExecutor` 所有退出点（成功/失败/预算/超时）触发
- hookManager 贯穿 ToolContext → spawnAgent / task / cowork / DAGScheduler / parallelCoordinator

**Part B — TaskCreated / TaskCompleted**:
- AgentTask 的 `onHook` 回调接入 HookManager
- 在 `register()`, `stop()`, `fail()`, `cancel()` 时触发

**Part C — PermissionRequest**（决策型 hook）:
- 在 toolExecutor 的 `needsUserApproval` 块中触发
- 可阻止权限请求（在弹出用户确认之前）
- 错误 fallthrough 到正常审批流程

相关代码：
- `src/main/tools/multiagent/spawnAgent.ts`
- `src/main/tools/multiagent/task.ts`
- `src/main/tools/toolExecutor.ts`

### 5. 新增 3 个 Observer-only 事件

| 事件 | 触发位置 | 用途 |
|------|----------|------|
| PermissionDenied | toolExecutor (3 条拒绝路径) | 审计安全策略拦截 |
| PostCompact | autoCompressor (压缩完成后) | 监控上下文压缩 |
| StopFailure | runFinalizer (circuit breaker + 超限) | 监控异常停止 |

附加：trigger history buffer（最近 50 条触发记录）用于 /hooks 可观测性。

相关代码：
- `src/main/hooks/hookManager.ts` — 触发方法 + history buffer
- `src/main/tools/toolExecutor.ts` — PermissionDenied 发射点
- `tests/unit/hooks/phase3.test.ts` — 13 个新增单元测试

### 6. /hooks 命令

展示当前 hook 配置、活跃事件监听、最近触发历史。修复了 AgentLoop/CLIAgent 未暴露 `getHookManager()` 导致命令获取 null 的问题。

相关代码：`src/shared/commands/definitions/newCommands.ts`

## Stream B: 控制面板命令（3 commits）

### 1. /cost — Token 用量与费用

显示当前会话的 token 使用量、按模型分列的费用统计、预算状态。

### 2. /context — 上下文窗口

显示上下文窗口占用率分解（系统/记忆/对话/工具结果）、压缩统计。

### 3. /status 增强

追加费用摘要行和上下文使用率行。

### 4. /agents 增强

历史记录条目追加费用估算。

### 5. /permissions — 安全决策可观测性

新增 `DecisionHistory` 服务（循环缓冲区，最多 50 条），记录 toolExecutor 中 10 个决策出口的每次权限决策结果。

/permissions 输出三部分：
1. 当前权限模式
2. Exec Policy 规则（已学习的模式）
3. 最近权限决策历史

决策类型：auto-approve / ask-approved / ask-denied / policy-allow / policy-deny / classifier-deny / hook-blocked / monitor-blocked

相关代码：
- `src/main/security/decisionHistory.ts` — DecisionHistory 服务
- `src/shared/commands/definitions/newCommands.ts` — 命令定义

## Stream D: 会话连续性

### 1. Git Branch 追踪

Session 类型新增 `gitBranch` 字段 + DB migration。会话创建时自动通过 `git rev-parse` 检测当前分支。

### 2. /sessions 增强

输出新增：分支名、PR 关联、工作目录、相对时间显示。

### 3. /resume 命令

注入历史会话上下文到当前会话，支持跨会话断点续传：
- 无参数：自动查找最近一个未完成会话
- 带 ID：加载指定会话上下文

相关代码：
- `src/shared/types/session.ts` — gitBranch 字段
- `src/main/services/core/repositories/SessionRepository.ts` — DB migration
- `src/main/services/infra/sessionManager.ts` — 分支检测
- `src/shared/commands/definitions/sessionCommands.ts` — /sessions, /resume
