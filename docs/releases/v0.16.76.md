# v0.16.76 — Plugin 体系收口 + Capability 路由 + 自动更新验证

> 48 commits since v0.16.75 | 2026-05-19

v0.16.75 首次跑通 Apple Developer ID 签名 + Notarization 之后的第一版 patch。
重点是把 PluginAPI v2 + 7 个 builtin plugin 收尾，引入 Marvis 风格的能力路由评估，
顺便验证自动更新链路。

---

## Plugin 体系（v2）收尾

### PluginAPI v2 + 7 个 builtin plugin

把媒体类工具按"独立 plugin"剥离，每个 plugin 自带 schema + handler + 平台约束：

| Plugin | 类型 | 覆盖工具 |
|---|---|---|
| `imageProcess` | 跨平台 | 图像基础处理 |
| `audioProcessing` | 跨平台 | 音频处理 |
| `videoGeneration` | 跨平台 | 视频生成 |
| `imageCreation` | 跨平台 | 文生图 |
| `browserControl` | 跨平台 | CDP 浏览器控制 |
| `computerUse` | macOS | 5 工具 + vision-only 模式 |
| `photoArchive` | macOS | Photos.app 归档 + 自动打标 |

衍生：
- 框架新增 `platforms/` 维度声明
- 抽 `imageGenerationService`，解 illustrationAgent ↔ imageGenerate 耦合
- `visionAnalysisService` 走 ModelRouter，分发零硬编码

### Capability 体系（Step 7）

引入"能力" (capability) 这一新维度，让模型/Plugin/Skill 三者通过 capability 互查：

- **PR 1**: capability schema + 模型能力单一权威 helper
- **PR 2**: `CapabilityRecommender` service + `recommend_capability` tool
- **PR 3**: GapCard UI + capability gap notice flow
- 配套：8 个 `workflow_recipe` 卡片对应新 builtin skill

### Marvis-inspired 路由评估

新增 capability assessments 框架，把"是否需要 vision"、"是否需要 web"等判断
从硬编码 if/else 改成基于评估器组合，方便后续动态扩展。
