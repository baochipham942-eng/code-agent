# v0.16.77 — Updater 诊断 + Admin Menu 回填

> 2026-05-19

修两个 v0.16.76 实测暴露的问题。

## Updater install 错误诊断

v0.16.76 通过 Tauri plugin-updater 安装时报"更新安装失败"，但 stdout 只有 callback
"Download finished, installing..."，错误从 `map_err` 直接吃掉给 IPC，看不到根因。
本版在 `install_update` 命令里把 `download_and_install` 返回的 error 用 `eprintln!`
完整打到 stderr，含 `Debug` 输出 + `Error::source()` 全链路 cause。

下次 install 失败时，在 terminal 启动 app 就能看到具体哪步炸：
- verify_signature 失败 → pubkey 不匹配 / sig 文件被破坏
- tar archive 解压失败 → entry 路径或权限问题
- fs::rename 失败 → /Applications 写权限 / 跨 fs

## Admin 菜单消失

v0.16.76 首启动时管理员菜单不显示。根因：Supabase profile fetch 5s timeout
（被并行 updater 下载抢带宽），fallback 到 basic user 返回 `{id, email}` 不带 `isAdmin`，
admin 入口条件挂掉。

本版改 `fetchUserProfile` 的 catch 分支，按优先级降级：
1. 内存里 `this.currentUser`（之前已 fetch 成功的）
2. 磁盘 secureStorage 缓存（含 isAdmin 字段）
3. 兜底 `{id, email}` 基本用户

之前只查内存缓存，磁盘缓存被忽略 → 冷启动场景必丢 isAdmin。
