# v0.16.78 — webServer mainWindow 注册修复

> 2026-05-19

修 v0.16.77 实测后发现的最根本问题。

## 根因

webServer 模式下创建了 `webModeWindow = new BrowserWindow()`（platform bridge stub，
`webContents.send` 走 SSE 推到 renderer），但**没把它注册到 `src/main/app/window.ts`
里 module-level 的 `mainWindow` 变量**。

结果：`initializeBackgroundInfra` line 409 `const mainWindow = getMainWindow()` 拿到
`null`，所有后台服务 callback 里的 `if (mainWindow) mainWindow.webContents.send(...)`
判断**永远 false**，事件全进黑洞。

直接受害者：
- AUTH 事件 → renderer authStore 永远不更新 → admin 菜单消失
- MCP 状态、Update 通知、Cron 触发等所有走 mainWindow.webContents.send 的 IPC

## Fix

`src/web/webServer.ts:480` 创建 webModeWindow 后立即：

```ts
import('../main/app/window').then(({ setMainWindow }) => {
  setMainWindow(webModeWindow as never);
});
```

让所有 `getMainWindow()` 调用都拿到 mock window，事件走 broadcastToRenderer → SSE。

## 顺带

继续带 v0.16.77 的 Rust 端 `eprintln!` updater 诊断 + authService fallback 兜底磁盘缓存。
本版升级时 install 如再失败，stderr 能看到具体 root cause。
