# Agent Neo v0.16.99

发布日期：2026-06-08

## 主要变更

### 启动稳定性
- 修复 notarized app 在慢启动环境下可能因 webServer 初始化超过 30 秒而被 Tauri healthcheck 杀掉的问题；启动等待窗口调整到 90 秒，覆盖 Supabase session timeout、旧库迁移和首次插件初始化的组合耗时。

### 模型配置与连接测试
- Claude 连接测试不再使用废弃的 `claude-3-haiku-20240307`，会优先使用当前选择模型；未选择时使用 provider 注册表里的当前默认模型。
- Provider/model fallback 对齐到当前 provider，避免 Claude provider 搭配 MiMo model 之类的错位状态。
- 发送前检查当前默认 provider 是否真正可用，避免“只配置了 Claude，但默认仍残留 MiMo”时继续打到错误 provider。

### 工具调用稳定性
- 修复 AI SDK 流式工具调用中已累积参数被终态空 input 覆盖的问题，降低 Write 工具收到空 `file_path/content` 的概率。
- 工具参数 schema 校验失败现在会带字段级 metadata，replay 可直接显示缺失字段和类型期望。

### 诊断与遥测
- `model_decision` 记录请求 provider/model、实际解析 provider/model、路由原因和 billing mode。
- 工具失败新增 `tool_args_validation` 分类，并低敏上报工具名、错误分类和校验失败摘要。
- structured replay 的 model/tool block 可回放模型决策和工具校验 metadata，减少靠截图判断问题。

### Release Gate
- PR 阶段新增 renderer bundle capability 校验，提前发现热更新 manifest 能力差异。
