# Agent Neo v0.17.2

发布日期：2026-06-18

## 主要变更

### 默认联网数据层
- **Firecrawl 成为默认 web 数据层**：`WebSearch` 和 `WebFetch` 对公网网页优先走 Firecrawl 搜索/抓取，免 key 可试用，配置 `FIRECRAWL_API_KEY` 后可获得更稳定额度。
- **保留原生兜底**：本地、私网、`.local`、`.internal` 和 JSON/CSV/XML/YAML 等原始数据 URL 继续走 native fetch；Firecrawl 不接触内网地址。
- **搜索源更可发现**：已配置但本轮未被智能路由使用的 Perplexity、Exa、Brave、Tavily 会给出轻提示，方便用户用 `sources` 指定检索源。

### 输入区降噪
- **推荐条更克制**：Skill 推荐最多展示 2 个，能力建议最多展示 3 个，并过滤 Skill/能力重复芯片。
- **搜索模型建议更准**：有工具调用能力的模型会被视为可通过 `WebSearch` 联网，不再误报“当前模型不擅长搜索”。

### 发版验收增强
- **新增发布后生产验收器**：`release:post-publish` 检查更新 API、下载跳转、落地页版本位、renderer rollout、OSS manifest、`release-record.json`、回滚状态和可选 Vercel 日志。
- **发版流程更清晰**：`release:neo --post-publish-verify` 可以在 CI 发布完成后直接跑只读线上验收，不会创建 tag 或上传 artifact。

## 修复
- 修复 Firecrawl keyless 额度耗尽时缺少可操作提示的问题，现在会引导配置 `FIRECRAWL_API_KEY`。
- 修复 Firecrawl 连续网络失败时每次请求都先吃满超时的问题，三次失败后短时冷却并走兜底。
- 修复非流式 OpenAI-compatible / Claude 工具调用会丢失前导正文和顺序信息的问题，工具块不再倒挂到答案下方。
- 修复打包运行时 Tauri 壳层和 Node webServer 日志目录不一致导致诊断导出难对齐的问题。
