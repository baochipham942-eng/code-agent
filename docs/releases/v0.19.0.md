# Agent Neo v0.19.0

发布日期：2026-06-22

## 主要变更

### 设计模式：多模型生图 + 视频生成
- **生图模型可切换**：设计模式文生图支持在 通义万相 / CogView-4 / FLUX.2 / **gpt-image-2** 间切换，基于能力标签化的视觉模型注册表；切换器只列「已配置 Key 且为视觉生成」的模型，聊天模型自动过滤。gpt-image-2 经自定义 OpenAI 兼容端点接入，文字/UI 渲染更适合设计稿。
- **视频生成（净新）**：设计画布支持文生视频（t2v）与图生视频（i2v），接入 通义万相视频 与 MiniMax 海螺双 provider，视频作为一等画布节点挂入非破坏性版本底座，生成前按时长显著提示预估成本。
- **标注重绘（非 wanx 模型整图编辑）**：在画布上圈选/标注（笔/箭头/矩形/文字）后，将标注烘入截图交由模型重绘并去除标注，给不支持 mask 的模型（如 gpt-image-2）开出免 mask 的编辑路径。

### 设计模式：对标能力补全
- **品牌/设计系统复用**：固化「我的品牌」色板、字体与组件 token 并在后续生成中强制注入，提升跨生成的一致性。
- **就地文本编辑**：交互原型支持点选文本直接编辑（无需重新生成）。
- **导出增强**：新增 PDF 与 PPTX 导出。

### Agent 执行引擎
- **执行引擎兼容矩阵**：新增 Agent 执行引擎能力矩阵与设置页执行引擎配置区。
- **MiMo-Code / Kimi Code 引擎集成**：接入 MiMo-Code 与 Kimi Code 执行引擎。

## 修复与加固
- 新增图片 URL 下载的 SSRF 守卫：仅允许 https 公网地址，拒绝私网/环回/元数据地址（修复 IPv6 字面量绕过的死代码缺陷）。
- 设计图各 IPC 在付费调用前拦截空白/越界参数，避免无效付费调用。
- 新增设计画布 IPC 能力均已登记 shell 能力清单，保证 renderer 热更新能力闸通过。
- 稳定性批次修复若干执行链路问题。
