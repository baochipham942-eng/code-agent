# Agent Neo v0.26.0

2026-07-12

## 用户可感知变化

- 长任务、Agent Team、Auto Agent、Multiagent 和动态工作流现在共享一套 durable 执行协议。进程退出或应用重启后，Neo 会依据持久化 checkpoint 和副作用证据恢复，减少任务无故丢失或整段重跑。
- Codex、Claude 等外部 CLI 运行会持久化 lifecycle 与 resume 证据；具备完整恢复信息时继续执行，证据不完整时转人工复核。
- MCP 异步任务可以按 provider task handle 继续查询；工具结果缓存只有在身份、能力和结果证据匹配时才会复用。
- Graph Runner 统一承接任务 DAG、动态工作流、外部引擎和子 Agent，既有事件消费者由集中 compatibility sink 保持兼容。

## 稳定性与恢复能力

- Durable Run Kernel 统一管理 run/operation 状态、owner epoch、lease、checkpoint、terminal uniqueness 和恢复分发。
- 启动期动态恢复与生产读路径已切换到 durable 状态，并通过 rollback round-trip 验证保留受控回退能力。
- 未知写副作用、外部 CLI resume 证据缺失、工作区或模型工具漂移不会盲目重放，而是进入 `requires_review`。
- OTel trace context 贯穿 run、operation、tool 和恢复链，便于定位跨进程执行问题。
- 连续按两次 ESC 的取消请求在同步入口去重，避免重复 cancel fan-out。

## 安全边界变化

- 文件、目录以及尚不存在目标通过 symlink 指向工作区外时，权限分类器都会要求确认。
- Seatbelt host 实测 5/5 通过；permission classifier 实测 20/20 通过；三类 symlink 越界写入边界均返回 `ask`。
- 可恢复工具调用仍以副作用证据为边界。无法证明结果或幂等性的写操作不会自动重复执行。

## 兼容性说明

- 没有新增必须迁移的用户配置，也没有已知的公开命令或文件格式破坏性变更。
- Durable run 数据库迁移由应用启动流程执行；v0.26.0 的生产读路径会优先读取新 durable 状态。
- 旧事件消费者继续通过 GraphEvent compatibility sink 接收兼容事件，但内部执行和恢复语义已统一到 Graph Runner。

## 已知限制

- 外部 CLI 不提供完整 resume 证据时，任务会停在人工复核状态，无法保证自动续跑。
- 未知副作用的 MCP/工具写入不会自动重试，需要用户判断是否继续。
- 工作区、模型或工具能力发生漂移时，durable run 会保护性暂停；这是避免在不同执行条件下错误续跑的预期行为。

## 升级与回滚

- 正式升级只通过 stable 更新通道与 v0.26.0 签名资产进行；安装前应结束正在运行的 Agent Neo 任务。
- 发现 P0/P1 问题时，应先停止 v0.26.0 的 stable 分发，再恢复上一稳定版本 v0.25.1 的 stable/update manifest，或发布新的 hotfix 版本。任何回滚动作都需要单独授权。
- 不覆盖用户数据库进行回滚。涉及 durable schema 的降级必须先保留数据副本，并使用发布流程记录的 rollback round-trip 路径。
