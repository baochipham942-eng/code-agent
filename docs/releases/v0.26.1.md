# Agent Neo v0.26.1

2026-07-12

## 用户可感知变化

- 本版本恢复 v0.26.0 被阻断的正式发布链。产品能力与 v0.26.0 保持一致，没有新增用户操作或配置迁移。
- renderer 热更新产物在 main 与 tag workflow 同时触发时保持同一个 bundle hash，避免桌面正式包因 manifest 与 release record 不一致而停止发布。

## 稳定性与发布安全

- renderer bundle 归档固定文件顺序、时间戳、所有者元数据和 gzip header；相同源码的重复构建会得到相同 SHA-256。
- 同一 renderer release channel 只允许一个 OSS publisher 写入，已经开始替换对象集的任务不会被新任务中途取消。
- OSS 先写 bundle 与 release record，最后写 manifest。客户端入口只在配套对象就绪后切换。
- 原有 renderer signature、bundle SHA-256、release-record、control-plane 和正式桌面发布门禁全部保留，没有降低校验或 freshness 标准。

## 兼容性说明

- 与 v0.26.0 的用户配置、durable run 数据和公开命令兼容。
- v0.26.0 的功能变更、durable recovery 与安全边界继续适用，详见 `docs/releases/v0.26.0.md`。
- 这是发布基础设施补丁，不改变 renderer 热更新协议字段。

## 已知限制

- 已存在的 `v0.26.0` tag 和失败 CI 现场保持不变；该版本没有 GitHub Release，不能作为完成发布使用。
- v0.26.1 只有在双架构正式 CI、GitHub Release、updater、renderer/control-plane、下载产物和生产日志审计全部通过后才能标记为生产 PASS。

## 升级与回滚

- 正式分发必须使用新的 `v0.26.1` annotated tag，不覆盖、删除或重发 `v0.26.0`。
- 上一稳定版本仍为 v0.25.1。任何停止分发、恢复 stable manifest 或回滚动作都需要单独授权。
- 不覆盖用户现有安装，不使用用户真实数据目录做 packaged smoke。
