# Agent Neo v0.26.2

2026-07-12

## 用户可感知变化

- 新安装或 fresh-profile 首次启动时，Agent Neo 不再等待所有远程 plugin、skill 和 MCP server 完成连接后才打开主窗口。
- 桌面壳会先完成本地配置、持久化、HTTP 健康检查和首窗导航；远程能力随后在后台初始化。
- 远程能力暂时不可用或连接失败时，内置界面仍可打开，受影响能力保持明确不可用，不会把未就绪状态误报为可恢复运行。

## 稳定性与发布安全

- Durable recovery 等 capability bootstrap 完成后再启动，并在 handler 未就绪前保持 `ready=false`，避免恢复任务过早 claim。
- `/api/update` 只接受 GET / HEAD，版本、release notes 和安装资产由 GitHub Release 与 OSS stable manifests 提供，不再接受无持久化效果的 Cloud publish payload。
- 发布后验证要求 update health 明确返回 `source=github_releases`，检测到其他 metadata source 会 fail-closed。
- 新增 repository-structure gate 与根目录设计契约，降低后续 Agent 修改跨层漂移和文档入口失配风险。

## 兼容性说明

- 与 v0.26.1 的用户配置、会话数据、Durable Run 数据和公开命令兼容，不需要数据迁移。
- 更新检查响应字段和 OSS stable manifest 结构保持不变；移除的是未持久化 metadata 的 POST 兼容入口。
- `onnxruntime-vad` 与 `playwright-browser-runtime` 继续作为可选资源，不进入桌面壳启动硬门。

## 已知限制

- `/api/health` 只证明桌面壳、本地 webServer 和首窗导航链可用，不代表所有远程 plugin、skill、MCP 或 Durable recovery 已完成初始化。
- 远程 capability 的实际可用时间仍取决于网络、认证状态和服务端响应；失败会记录 warning 并保持对应能力不可用。
- 只有双架构正式 CI、GitHub Release、OSS stable manifests、renderer/control-plane、安装资产和发布后探针全部通过，v0.26.2 才能标记为生产 PASS。

## 升级与回滚

- 正式分发使用新的 `v0.26.2` annotated tag，不覆盖或移动既有 tag。
- 回滚需要同时恢复同一上一稳定版本的 `stable/latest.json` 与 `stable/release.json`，并单独核对 renderer rollout；不删除既有 Release，不覆盖用户数据。
- packaged smoke 使用隔离 HOME 和数据目录，不读取或修改用户现有安装数据。

## 修复证据

- 定向回归覆盖发布门、发布后验证、update metadata、Durable recovery 启动顺序和 Web capability bootstrap。
- source-side release gates 必须在 release commit 上通过；正式 CI 与发布后证据在 tag workflow 完成后补齐。
