# Agent Neo v0.26.3

2026-07-12

## 用户可感知变化

- 新安装或 fresh-profile 首次启动时，Agent Neo 不再等待所有远程 plugin、skill 和 MCP server 完成连接后才打开主窗口。
- 桌面壳先完成本地配置、持久化、HTTP 健康检查和首窗导航，远程能力随后在后台初始化。
- 远程能力暂时不可用或连接失败时，内置界面仍可打开；受影响能力保持不可用，Durable recovery 在依赖未就绪时保持 fail-closed。

## 稳定性与发布安全

- App-host Stability smoke 使用隔离数据目录，不读取用户真实 MCP、skill、会话或数据库配置。
- fresh Stop/Recovery 证据绑定 commit `72f261f4695b91730303da60990f6fcd02a8ea2d`：RunRegistry 53ms 释放、renderer Stop 84ms 清理、Bash cancel 58ms、HTTP cancel 1ms，全部完成终态清理。
- `/api/update` 只接受 GET / HEAD，版本、release notes 和安装资产由 GitHub Release 与 OSS stable manifests 提供；发布后验证拒绝非权威 metadata source。
- repository-structure gate 与根目录设计契约进入持续检查。

## 兼容性说明

- 与 v0.26.1 的用户配置、会话数据、Durable Run 数据和公开命令兼容，不需要数据迁移。
- 更新检查响应字段和 OSS stable manifest 结构保持不变；移除的是未持久化 metadata 的 POST 兼容入口。
- `onnxruntime-vad` 与 `playwright-browser-runtime` 继续作为可选资源，不进入桌面壳启动硬门。

## 已知限制

- `/api/health` 证明桌面壳、本地 webServer 和首窗导航链可用，不代表所有远程 plugin、skill、MCP 或 Durable recovery 已完成初始化。
- 远程 capability 的实际可用时间仍取决于网络、认证状态和服务端响应；失败会记录 warning 并保持对应能力不可用。
- `v0.26.2` tag 已存在，但 macOS CI 在打包前被旧 Stability evidence 阻断，没有 GitHub Release、安装资产或 stable promotion；该 tag 保留作为失败现场，不移动、不覆盖。
- 只有双架构正式 CI、GitHub Release、OSS stable manifests、renderer/control-plane、安装资产和发布后探针全部通过，v0.26.3 才能标记为生产 PASS。

## 升级与回滚

- 正式分发使用新的 `v0.26.3` annotated tag，不覆盖、删除或重发 `v0.26.2`。
- 回滚需要同时恢复同一上一稳定版本的 `stable/latest.json` 与 `stable/release.json`，并单独核对 renderer rollout；不删除既有 Release，不覆盖用户数据。
- packaged smoke 使用隔离 HOME 和数据目录，不读取或修改用户现有安装数据。
