# Agent Neo v0.26.4

2026-07-12

## 用户可感知变化

- 新安装不再把语音输入和网页操作组件算作桌面壳缺失资源。它们会在第一次使用对应能力时按需下载。
- arm64 首次开启本地音频只安装语音输入组件；首次调用浏览器自动化只安装网页操作组件，避免一次下载全部可选组件。
- 更新设置对普通用户展示本机组件状态、下载进度和失败重试。Intel Mac 的语音 VAD 显示“不适用”，网页操作组件仍可安装使用。

## 分发与信任链

- macOS arm64 发布 `onnxruntime-vad` 与 `playwright-browser-runtime`；macOS x64 发布 `playwright-browser-runtime`。
- 每个架构分别生成签名 manifest、manifest sha256 和独立 archive，上传后立即从 OSS 回读验证签名、摘要、平台和最小资产数。
- `stable/release.json` 同时包含 arm64 与 x64 runtime-assets manifest/sha sidecar，`/api/update` 按 `arch` 返回对应 metadata。
- renderer manifest 默认声明两项 runtime asset；x64 的 VAD 由平台状态标记为 unsupported，不触发下载也不阻断热更新。

## 诊断与发布门禁

- fresh install 允许可选组件处于“首次使用时下载”，只有包内必需资源缺失才判桌面壳失败。
- 首次安装完成后，active runtime manifest 和诊断状态会切换为 installed。
- 正式 tag 的双架构 runtime-assets 构建不可跳过；签名密钥缺失、上传验证失败或任一架构 metadata 缺失都会阻断发布。
- 发布后验证分别请求 arm64 与 x64 更新检查，要求两侧都返回 manifest URL 与 sha256。

## 兼容性与回滚

- 不修改、不移动 `v0.26.3` tag，也不替换其 GitHub Release 或 OSS 版本化资产。
- 与 v0.26.3 的用户配置、会话和 Durable Run 数据兼容，不需要数据迁移。
- 回滚时同步恢复上一稳定版本的 `stable/latest.json` 与 `stable/release.json`，并核对 renderer rollout；已安装的版本化 runtime asset 可保留，active manifest 仍受签名和哈希校验约束。

## 发布验收

- 源侧门禁、双架构 workflow 静态测试、runtime installer/状态/renderer 依赖测试必须通过。
- 正式 CI 后需确认 GitHub Release、OSS versioned runtime-assets、`stable/release.json`、arm64/x64 `/api/update` 和首次安装后的 installed 诊断。
