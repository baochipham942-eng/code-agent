# Agent Neo v0.27.0

2026-07-14

## 用户可感知变化

- 长输出任务的停止链路完成端到端收敛：停止按钮会在 run 受理后出现，重复停止不会失效，界面只有在 registry 或 durable run 真正进入终态后才显示已取消。
- 新会话创建期间发送消息会等待新会话绑定完成；创建失败时会建立新的安全目标，不再把消息送回用户刚离开的旧会话。
- 设置页新增官方 Skill 市场货架、新版提示和手动升级；安装固定到来源 commit，并校验内容哈希。项目可以覆盖全局 Skill 启用状态。
- 模型设置按真实能力展示推理深度、思考开关或预算控制，减少不支持参数造成的调用失败。
- Mermaid 支持标注后直接编辑、缩放和平移；长会话中的已完成 Markdown 内容减少离屏渲染开销。
- 记忆召回在正则快速路径之外增加分类器判断，提高自然表达下找回历史上下文的命中率。

## 稳定性与安全

- AbortController 在 run 受理时武装；取消请求按同一 runId 串行重投递，HTTP、renderer、registry、durable 状态使用同一终态证据。
- CLI serve 取消入口不再只返回成功文本，会真实投递 AgentLoop cancel 并以 `task_cancelled` 结束流。
- 主任务、队友、异步 Agent、cron 与后台 spawn 接入统一权限拓扑；需要确认的工具调用不能被子路径绕过。
- ADR-038/039 完成运行时状态 owner 收敛，统一 repair 无进展逃生、降级收尾与 trace/artifact 生命周期。

## 兼容性与回滚

- 与 v0.26.4 的用户配置、会话和 Durable Run 数据兼容，不需要手工迁移。
- 回滚时恢复 v0.26.4 的 `stable/latest.json` 与 `stable/release.json`，并同步核对 renderer rollout；版本化发布资产不覆盖旧版本。

## 发布验收

- 源侧 release gates、Knip ratchet、typecheck、全量 Vitest、PR WebServer/renderer/Swarm smoke 与 Swarm e2e 必须通过。
- 正式 tag CI 必须完成 macOS arm64/x64 签名与公证、renderer release gate、runtime-assets 双架构发布、GitHub Release 和 OSS stable 提升。
- 发布后必须核对 GitHub Release 资产、`stable/latest.json`、`stable/release.json`、arm64/x64 更新 API 与 renderer rollout 一致性。
