## [19:12] v0.16.35 打包安装 — N-API 崩溃修复

### 问题
- Electron 启动后崩溃（SIGABRT），栈帧：`Napi::AsyncWorker::OnWorkComplete` → `ThrowAsJavaScriptException`
- 原因：原生模块（isolated-vm / better-sqlite3 / keytar）未针对 Electron 38 重编译，ABI 不匹配

### 修复
1. `npm run rebuild-native` — 三个原生模块全部重编译成功
2. `npm run dist:mac` — 打包 Code Agent-0.16.35-arm64.dmg
3. 安装后启动正常，无崩溃

### 教训
**每次 `npm install` 或依赖变更后，打包前必须先 `npm run rebuild-native`**。npm install 安装的原生模块是针对系统 Node.js 编译的，与 Electron 的 Node ABI 版本不同，直接打包会导致运行时 N-API 崩溃。虽然 postinstall 钩子会自动执行，但如果跳过了（如 `--ignore-scripts`），必须手动重编译。
